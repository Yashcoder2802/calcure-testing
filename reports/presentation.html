What is Calcure?

A Terminal User Interface (TUI) calendar and task manager that runs entirely in the terminal — no GUI, no mouse, just your keyboard.

  • Create & manage calendar events
  • Track to-do tasks with deadlines
  • Set up recurring events (daily / weekly / monthly / yearly)
  • Import standard .ics calendar files
  • Built-in task timer (Pomodoro-style)
  • Supports Persian calendar conversion
Open Source Python 3.9+ Linux / macOS ~1,100 GitHub stars

Blackbox Testing — Four Techniques

EP
Equivalence Partitioning — divide inputs into valid/invalid classes, one test per class
E.g. task name: empty · 1–999 chars · ≥1000 chars · whitespace-only
BA
Boundary Value Analysis — test at the exact edge of every partition
E.g. name length 999 (valid) vs 1000 (invalid); timer format switches at exactly 3600 s
EG
Error Guessing — intuition-driven corner cases a real user might hit
E.g. delete non-existent ID · rename to empty string · pass None as index · malformed CSV
CT
Combinatorial Testing — pairwise combinations of multiple parameters
E.g. frequency × repetition (8 combos) · status × privacy (4 combos) · firstweekday × all 12 months
117 tests 10 test classes 2 fault-revealing failures No source code used to write tests

Blackbox — Example Tests

Boundary Analysis — Name Length

def test_add_task_999_char_name_accepted(self):
    """[BA] 999 chars — exactly within valid range."""
    self.tasks.add_item(make_task(name="x" * 999))
    self.assertEqual(len(self.tasks.items), 1)

def test_add_task_name_over_1000_chars_rejected(self):
    """[BA] 1000 chars — first invalid length."""
    self.tasks.add_item(make_task(name="x" * 1000))
    self.assertEqual(len(self.tasks.items), 0)

Combinatorial — Frequency × Repetition

def test_ct_weekly_rep2_one_extra_date(self):
    """[CT] WEEKLY × rep=2 → 1 extra occurrence."""
    self.assertEqual(
        len(self._make_repeated(2, Frequency.WEEKLY).items), 1)

def test_ct_daily_rep5_four_extra_dates(self):
    """[CT] DAILY × rep=5 → 4 extra occurrences."""
    self.assertEqual(
        len(self._make_repeated(5, Frequency.DAILY).items), 4)

Error Guessing — Edge Cases

def test_delete_nonexistent_task(self):
    """[EG] Delete unknown ID — safe & silent."""
    self.tasks.add_item(make_task(item_id=1))
    self.tasks.delete_item(99)
    self.assertEqual(len(self.tasks.items), 1)

def test_is_valid_number_none(self):
    """[EG] None input — must not crash."""
    self.tasks.add_item(make_task())
    self.assertFalse(self.tasks.is_valid_number(None))

Timer — Observable Output Only

def test_elapsed_time_format_hh_mm_ss_at_one_hour(self):
    """[BA] ≥60 min → HH:MM:SS format."""
    base = 1_000_000
    t = make_timer([base, base + 3660])
    self.assertRegex(t.passed_time, r"^\d{2}:\d{2}:\d{2}$")

Fault 1 — Whitespace-Only Task Names Accepted

Bug in calcure/data.py · Collection.add_item()

A name like "   " (three spaces) passes validation and creates an invisible task the user can never find.

The Faulty Guard

# Current code — BUGGY
if 1000 > len(item.name) > 0:
    # "   " has len=3 > 0 → ACCEPTED ❌
    self.items.append(item)

The Fix

# Correct guard — strip whitespace first
if 1000 > len(item.name.strip()) > 0:
    self.items.append(item)

Fault-Revealing Test

def test_add_task_whitespace_only_name_rejected(self):
    """[EG] Whitespace-only name — should be
    rejected like an empty name.
    FAULT: len("   ") = 3 > 0 so it passes."""
    self.tasks.add_item(make_task(name="   "))
    self.assertEqual(len(self.tasks.items), 0)
    # ❌ FAILS → confirms the bug
Severity

Medium — user creates invisible tasks · name search returns nothing · no error shown

Fault 2 — Malformed CSV Crashes the App

Bug in calcure/loaders.py · EventLoaderCSV.load()

If the events CSV has a non-integer year field (manually edited or corrupted), calcure crashes on startup with an unhandled ValueError.

The Faulty Line

# loaders.py:114 — no try/except
year = int(row[1])
# ValueError: invalid literal for int()
# with base 10: 'not-a-year'  ❌

The Fix

try:
    year = int(row[1])
except ValueError:
    continue  # skip bad row silently

Fault-Revealing Test

def test_event_loader_malformed_csv_does_not_crash(self):

    skipped gracefully, not crash.

Lessons Learned

1
Blackbox testing works without formal specs.

Calcure has no API docs. We derived partitions from function names, docstrings, and README — enough to find 2 real bugs.

2
UI code is genuinely untestable with unit tests.

~1,400 lines of curses code require a live terminal. CLI testing with PTY was the only way to exercise end-to-end behavior.

3
Mutation testing reveals gaps that coverage misses.

We had 89% branch coverage but still 144 surviving mutants — mostly relaxed comparators and optional-field defaults invisible to coverage tools.

4
Mocking is essential for non-deterministic dependencies.

Without mocking time.time(), timer tests are flaky. Without mocking urllib, ICS tests need internet access. Mocks make tests fast, isolated, and repeatable.
